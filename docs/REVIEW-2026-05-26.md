deterministic workflows over reasoning; minimize tokens needed to load the
operating context.**

## What was deliberately NOT changed (yet)

These are higher-risk edits worth a follow-up PR rather than batching here:

1. **Remove duplicated command tables** across `README.md`, `AGENTS.md`,
   `RUNBOOK.md`, `.github/copilot-instructions.md`. The cheatsheet now owns
   the canonical table; the others can shrink to one-liners with a link.
   *Risk:* breaks people's bookmarks; do it in a dedicated PR.

2. **Move "Iteration N Lessons" from `RUNBOOK.md`** into either
   `docs/dogtail-testing.md` (already canonical for dogtail) or
   `docs/archive/`. Currently mixes timeless architecture with one-time
   debugging history.

3. **Trim `.github/copilot-instructions.md`** — it currently duplicates
   AGENTS.md + cheatsheet content. Should become a thin pointer.

4. **Document titan-disk authorized_keys refresh path** or formally close
   it as human-gated escalation (currently §7 of cheatsheet just says
   "file an issue").

5. **Replace lab-operations §5 vague branches** ("usually a YAML schema
   break", "read the message") with strict expected-output decision trees.

## How to use the cheatsheet

Weak-agent load order:

1. `AGENTS.md` — policy only (short).
2. `docs/agent-cheatsheet.md` — operations.
3. Stop. Escalate to lab-operations / WORKFLOWS / RUNBOOK / dogtail-testing
   only when the cheatsheet explicitly links into them.

The cheatsheet is structured as lookup tables, not prose. Every row maps a
symptom or task to **exactly one** named recipe or command — no inference
required.
