How to Reproduce & Run Tests Locally

Bluefin operates as an atomic, image-based OS. You can replicate the exact dashboard BDD verification checks locally using the following steps:

  1. Verify Cluster Access: Run just list-workflows to confirm Argo connectivity and namespace visibility.
  2. Sync Workflow Definitions: Execute just argocd-sync if you need the latest GitOps state before testing.
  3. Container-Only Validation: Run just run-tests-tag testing or just run-tests-tag lts-testing to test the published bootc OCI image directly as a container.
  4. Matrix Smoke Check: Run just run-tests-matrix when you want the default testing and lts-testing smoke pair.

Core BDD Test Suites & Assertions

Each test lane executes targeted BDD scenarios written in Gherkin and backed by python-behave, dogtail, and GNOME AT-SPI tree traversal:

SuiteRuntimeKey System Assertions & Scenarios Verified
smokeContainerGNOME Shell Desktop Environment: Verifies AT-SPI registry daemon readiness, Quick Settings menus, Clock and Calendar top-bar responsiveness via Shell.Eval, Caffeine extension state, and Dash to Dock layout integrity.
commonContainerOS Integrations: Asserts flatpak sandbox portal boundaries, polkit local user authorization overrides, basic immutable host security policies, and standard desktop session persistence.
developerContainerDeveloper Toolchains: Confirms Homebrew prefix pathing, Podman rootless socket readiness, Ptyxis terminal tab multiplexing, and development build tools accessibility.

About the test split

Gating split (ADR 0002 amendment)

What the suites verify

A lane's release verdict is good only when build, signature verification, and the gating QA suites all pass. Gating suites prove the image-based, atomic platform contract. Informational suites are tracked, linked, and trended, but they do not block a release.

Desktop startup, system extensions, top-bar menus, and primary app wrappers.

systemGATE
Atomic OS Contract

bootc status, read-only /usr, staged upgrades, composefs, rollback behavior.

flatcarGATE
Flatcar VM

Flatcar node lifecycle and systemd/container contracts, gating only for the flatcar lane.

Flatpak portals, polkit overrides, and user-space isolation boundaries.
